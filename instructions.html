Scheduling Application Instructions

Note: This application helps you schedule tasks based on available resources and dynamic dependencies.

How to Use the Application

Step 1: Prepare Your Excel File

Create an Excel file with two sheets:

Sheet 1: Tasks

Format Sheet 1 as follows:

Task Name Priority BE API Android IOS QC Creation QC Execution
VCN - Listing 2 1 6 6 0.5 2
VCN - Creation 1 1 5 5 1 3
... ... ... ... ... ...

List all tasks with their respective priorities and time estimates (in days) for each stage.

Sheet 2: Resources

Format Sheet 2 as follows:

BE API IOS Android QC
George David Mohamed Abdullah
Bahey Sara Neyazy
Ezz

List all available resources under their respective categories.

Default Dependencies:
  • Tasks with "Creation" must be completed before tasks with "Execution" can start.
  • Tasks in the "BE API" category must be completed before tasks in "IOS" and "Android" categories can start.
  • Tasks in the "IOS" and "Android" categories must be completed before tasks in the "QC execution" category can start.
  • Tasks labeled "QC Creation" must be completed before tasks labeled "QC execution" can start.
  • The same resource should handle both "QC Creation" and "QC execution" tasks.

Step 2: Configure Dependency Rules

The application includes a Dependency Rules window that allows you to dynamically configure the dependencies between departments.

  • Accessing the Dependency Rules Window: Open the application and select the "Dependency Rules" option from the main menu.
  • Configuring Dependencies: In the Dependency Rules window, you'll see a list of departments along with checkboxes for other departments. For each department, select the checkboxes corresponding to the departments that must be completed before it can start.
  • Saving Changes: After configuring the dependencies, click the "Save Dependencies" button to store the rules. These will be saved in a JSON file (e.g., dependency_rules.json) and applied when generating the schedule.
  • Updating Dependencies: You can update the dependencies at any time by reopening the Dependency Rules window and making adjustments.
Example JSON file format: 
       { 
            "Creation": ["Execution"],
            "BE API": ["IOS", "Android"],
            "IOS": ["QC execution"],
            "Android": ["QC execution"],
            "QC Creation": ["QC execution"]
        }
  • Save Changes: Save the updated rules file and ensure it is in the correct location where the application can access it. (same path as the application)

Step 3: Launch the Application

Open the Scheduling Application on your computer.

Step 4: Select Input File

Click the "Browse" button next to "Input Excel File" and select your prepared Excel file.

Step 5: Choose Output File Location

Click the "Browse" button next to "Output Schedule File" to choose where you want to save the results.

Step 6: Run the Scheduler

Click the "Run Scheduler" button to process your data and generate the schedule based on the defined dependencies.

Step 7: Review Results

Once the process is complete, you'll see a success message, and the application will automatically open the output file.

Tip: Ensure your input file is correctly formatted to avoid errors. The scheduler will apply the dynamic dependencies and resource allocation rules as specified.
